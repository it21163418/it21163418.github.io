AI-Enhanced and Access Management System
For Internal Banking Systems

Project Scope

Research Gap

Current Identity and Access Management (IAM) systems used in Sri Lankan banking environments heavily rely on static credentials and fixed biometric mechanisms, which do not adapt to changing user behavior or contextual factors such as login location, device type, or session activity. These systems lack the ability to perform real-time risk assessments, making them insufficient for addressing evolving operational needs. Additionally, there is limited integration of advanced techniques like reinforcement learning, behavioral biometrics, and predictive analytics. Banks often struggle to implement adaptive security due to the absence of continuous authentication models and tools that analyze user behavior dynamically. Existing systems also do not monitor insider behavior effectively or provide predictive insights into system usage patterns, leaving a notable gap in intelligent, responsive IAM solutions.

Research Problem

This research addresses the problem of enhancing identity verification and session monitoring in internal banking systems through AI techniques. The focus is on how context-aware risk scoring and behavioral biometrics can improve authentication accuracy and operational integrity. The study seeks to understand how reinforcement learning models can be used for continuous authentication, how predictive analytics can forecast abnormal user behavior, and how clustering algorithms can be used to detect and respond to irregular usage patterns. The aim is to build a system that is more adaptive, intelligent, and capable of responding to contextual changes without negatively impacting user experience.

Literature Review

Academic literature points to the limitations of traditional IAM systems, which use static methods of user authentication and fail to capture behavioral or contextual data. Research has shown that behavioral biometrics such as typing patterns, mouse movement, and access consistency can improve user verification by providing continuous, non-intrusive monitoring. Reinforcement learning models like DDQN (Double Deep Q-Network) and PPO (Proximal Policy Optimization) are highlighted for their capacity to adapt to user behavior and improve authentication outcomes over time. Predictive models like Random Forest have been effective in analyzing historical data to anticipate future behavior, while clustering techniques such as K-means and DBSCAN help identify usage anomalies. These methods support the development of systems that are more responsive and tailored to operational needs in banking.

Research Objectives

To design and develop an intelligent, AI-supported Identity and Access Management (IAM) system tailored for internal banking systems in Sri Lanka. This system will move beyond static authentication by integrating real-time contextual awareness, behavioral monitoring, and adaptive security mechanisms to ensure both secure and seamless user experiences.

Multi-Dimensional Risk Scoring System

Implement a comprehensive risk scoring engine that assesses user login attempts by analyzing device type, session duration, IP address, login location, and historical behavior. The risk score will determine the appropriate action whether to permit, deny, or escalate access based on real-time context and previous user interactions.

Continuous Behavioral Monitoring

Incorporate behavioral biometrics such as typing speed, mouse movements, access patterns, and login consistency to provide continuous authentication beyond the initial login. This ensures that session integrity is maintained throughout the userâ€™s activity and enables real-time detection of unusual behavior.

Reinforcement Learning-Based Adaptive Authentication

Utilize reinforcement learning models, particularly DDQN (Double Deep Q-Network) and PPO (Proximal Policy Optimization), to train the system to adaptively authenticate users. These models will learn and evolve based on user behavior over time, increasing authentication accuracy and reducing reliance on static rule sets.

Predictive Analytics with Historical Data

Apply Random Forest machine learning algorithms to analyze historical user access logs. The goal is to predict potential behavioral deviations and preemptively identify risks before they escalate, thereby enabling the system to take preventive actions based on predictive trends.

Real-Time Contextual Data Integration

Leverage real-time data such as device fingerprinting, screen resolution, OS and browser information, geolocation (using the Haversine formula), and session timing. This data will be processed across various layers device, session, and geo-network to support context-aware decision-making and dynamic authentication workflows.

Methodology

Figure 1. High Level Architecture of the system.

The proposed methodology begins with the collection of historical and real-time user activity data, including device, session, and geo-network details. Context-aware authentication is achieved by calculating risk scores across three dimensions device and session, behavioral patterns, and geographic information. The system uses the Haversine formula to measure travel distance between login locations and flags impossible travel or unusual login behavior. Reinforcement learning models such as DDQN and PPO are trained on behavioral data to allow the system to adapt over time. A Random Forest model is developed to evaluate access logs and forecast unusual behavior. Insider activity is monitored through clustering algorithms like K-means and DBSCAN, as well as anomaly detection using Isolation Forests. The system is continuously updated and validated to maintain accuracy and responsiveness.
